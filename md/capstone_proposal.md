# 스마트 캠퍼스 포털 - 캡스톤 프로젝트 기획서

## 1. 프로젝트 개요

### 1.1 프로젝트명
**스마트 캠퍼스 포털 (Smart Campus Portal)**

### 1.2 한 줄 소개
RAG 기반 AI 챗봇과 PWA 기술을 활용한 차세대 스마트 캠퍼스 포털

### 1.3 개발 기간
2026년 3월 ~ 2026년 6월 (4개월)

### 1.4 팀 구성
- 2명 (프론트엔드 1명, 백엔드 1명)

---

## 2. 배경 및 필요성

### 2.1 현황 분석
현재 대부분의 대학 홈페이지는 다음과 같은 문제점을 가지고 있음:

**문제점**
- ❌ 정보가 여러 페이지에 분산되어 찾기 어려움
- ❌ 모바일 최적화가 부족함 (네이티브 앱 없음)
- ❌ 학사 정보를 찾기 위해 여러 번 클릭 필요
- ❌ 강의 선택 시 참고할 수 있는 정보 부족
- ❌ 졸업 요건 확인이 복잡함
- ❌ 익명 보장 없는 수강평 시스템 (신뢰도 낮음)

### 2.2 필요성
- 학생들이 필요한 정보를 **빠르고 쉽게** 찾을 수 있는 플랫폼 필요
- **AI 기술**을 활용한 스마트한 학사 정보 제공
- **모바일 환경**에 최적화된 UI/UX
- 학생들의 **실제 니즈**를 반영한 기능 구현

## 3. 프로젝트 목표

### 3.1 주요 목표
1. 학생 중심의 직관적이고 사용하기 쉬운 캠퍼스 포털 구축
2. AI 챗봇을 통한 즉각적인 학사 정보 제공
3. 졸업까지 필요한 학점 및 수강 과목 관리 기능
4. 강의 선택에 도움이 되는 리뷰 시스템 구축

### 3.2 기대 효과
- 학생들의 **정보 접근성 향상** (평균 검색 시간 70% 단축)
- 학사 행정 관련 **문의 감소** (AI 챗봇이 FAQ 자동 응답)
- 수강 신청 시 **의사결정 지원** (강의 리뷰 및 학점 계산기)
- **모바일 사용성 개선** (기존 대비 사용자 만족도 향상)

## 4. 주요 기능

### 4.1 기본 기능

#### 📱 PWA (Progressive Web App) ⭐ 핵심 차별점
- **홈 화면에 추가** 가능 (앱처럼 설치)
- **오프라인 모드** 지원 (Service Worker)
- **푸시 알림** (공지사항 알림)
- 앱 아이콘, 스플래시 화면 커스터마이징
- PC, 태블릿, 모바일 모든 기기에서 최적화된 UI
- 다크모드 지원

**해결한 문제**: "모바일 앱이 없다"는 문제를 네이티브 앱 개발 없이 해결

#### 👤 회원 관리
- 학생 회원가입 및 로그인
- 학번, 학과, 학년 정보 관리
- 마이페이지 (내 정보 수정)

#### 📢 공지사항 & 학사일정
- 카테고리별 공지사항 (학사/장학/취업/행사)
- 캘린더 형식의 학사일정 조회
- 중요 공지 알림 기능

---

### 4.2 핵심 기능 (차별화)

#### 🤖 AI 챗봇 (핵심 1) - RAG 기반
**기능 설명**
- 학사 정보에 대한 자연어 질의응답
- 강의실 위치, 학사일정, 졸업요건 등 즉각 답변
- **실시간 데이터 기반** (주기적 크롤링으로 최신 정보 유지)

**기술 구현** ⭐ 차별점
1. **웹 크롤링**: Puppeteer로 학교 홈페이지에서 공지사항 자동 수집
2. **데이터 전처리**: 텍스트 정제 및 청크 분할 (LangChain)
3. **벡터화**: OpenAI Embeddings API로 텍스트를 벡터로 변환
4. **벡터 저장**: MongoDB Atlas Vector Search에 저장
5. **RAG 파이프라인**:
   - 사용자 질문 → 벡터 변환
   - 유사도 검색 → 관련 문서 추출
   - Context + 질문 → GPT-3.5 → 정확한 답변

**크롤링 스케줄러**
- node-cron으로 매일 새벽 2시 자동 크롤링
- 새로운 공지사항만 업데이트 (중복 제거)

**예시 질문**
- "컴퓨터공학과 졸업 학점은 몇 학점이야?"
- "IT관 301호는 어디에 있어?"
- "이번 학기 수강신청 기간이 언제야?"

#### 📊 학점 계산기 & 졸업요건 체커 (핵심 2)
**기능 설명**
- 현재까지 취득한 학점 입력 → 자동 계산
- 전공/교양/자유선택 이수 현황 시각화
- 졸업까지 필요한 학점 및 과목 추천
- **성적표 PDF 업로드** → 자동 파싱 기능 (와우 포인트!)

**구현 범위** (현실화)
- **1차**: 컴퓨터공학과 (우리 학과) 정밀 지원
- **2차**: 타 학과는 사용자가 졸업요건을 커스텀 입력
- **이유**: 학번/학과마다 복잡한 요건을 모두 코딩하기엔 리소스 부족

**화면 구성**
- 원형 그래프: 전체 졸업 이수율
- 막대 그래프: 영역별 (전공/교양) 달성률
- 부족한 학점 및 필수 과목 리스트
- 성적표 업로드 버튼

#### ⭐ 강의 리뷰 시스템 (핵심 3)
**기능 설명**
- 수강 완료한 과목에 대한 리뷰 작성
- 별점 (1~5점), 태그 (#꿀강, #과제많음, #시험어려움)
- 강의 선택 시 다른 학생들의 리뷰 참고 가능
- **학교 이메일 인증** (@university.ac.kr) - 신뢰성 확보

**인증 프로세스** (에브리타임 방식)
1. 회원가입 시 학교 이메일 입력
2. Nodemailer로 인증 코드 발송
3. 인증 완료 후에만 리뷰 작성 가능
4. 익명성 보장 (이메일은 노출 안 됨)

**주요 항목**
- 강의 만족도 (5점 척도)
- 난이도, 과제량, 출석 체크 여부
- 한 줄 평 (익명 작성)
- 평점 통계 및 키워드 분석

## 5. 기술 스택

### 5.1 프론트엔드
```
- React 18.x (UI 라이브러리)
- TypeScript (타입 안정성)
- PWA (Progressive Web App) - 모바일 앱처럼 설치 가능
- TailwindCSS + shadcn/ui (모던 UI 컴포넌트)
- React Router (라우팅)
- Axios (HTTP 통신)
- Recharts (통계 시각화)
```

### 5.2 백엔드
```
- Node.js + Express (API 서버)
- TypeScript (타입 안정성 및 AI 코딩 최적화)
- MongoDB Atlas (클라우드 NoSQL + Vector Search)
- Mongoose (ODM)
- JWT (인증/인가)
- Nodemailer (이메일 인증)
```

### 5.3 AI & 데이터
```
- OpenAI API (GPT-3.5 + Embeddings)
- LangChain.js (RAG 파이프라인 구축)
- Cheerio / Puppeteer (웹 크롤링)
- node-cron (스케줄러 - 주기적 크롤링)
```

### 5.4 배포 & 도구
```
- 버전 관리: Git & GitHub
- 프론트엔드 배포: Vercel (PWA 지원)
- 백엔드 배포: Railway / Render
- API 문서: Swagger (자동 생성)
- API 테스트: Postman
```

## 6. 시스템 아키텍처

### 6.1 전체 구조
```
┌─────────────────┐
│  학교 홈페이지   │
└────────┬────────┘
         │ Crawling (Puppeteer)
         ↓
┌─────────────────────────────┐
│   MongoDB Atlas             │
│  ┌──────────────────────┐   │
│  │ 공지사항/학사일정    │   │
│  └──────────────────────┘   │
│  ┌──────────────────────┐   │
│  │ Vector Store         │ ← Embeddings (OpenAI)
│  │ (RAG용 벡터 검색)    │   │
│  └──────────────────────┘   │
└─────────────────────────────┘
         ↑
         │ API (REST)
         ↓
┌─────────────────────────────┐
│   Node.js API Server        │
│   (Express + LangChain)     │
└───────────┬─────────────────┘
            │ API
            ↓
┌─────────────────────────────┐      ┌─────────────┐
│   React PWA 앱              │ ←──→ │ OpenAI API  │
│   (TypeScript)              │      │ (GPT-3.5)   │
└─────────────────────────────┘      └─────────────┘
         ↑
         │ 설치 (홈 화면에 추가)
         ↓
┌─────────────────┐
│  사용자 (학생)   │
└─────────────────┘
```

### 6.2 RAG 파이프라인 (AI 챗봇 핵심)
```
1. [데이터 수집]
   학교 홈페이지 크롤링 → 텍스트 추출
   
2. [벡터화]
   OpenAI Embeddings API → 텍스트를 벡터로 변환
   
3. [저장]
   MongoDB Atlas Vector Store → 벡터 저장
   
4. [질문 처리]
   사용자 질문 → 벡터 변환 → 유사도 검색
   
5. [답변 생성]
   검색된 관련 문서 + 질문 → GPT-3.5 → 답변
```

## 7. 데이터베이스 설계 (초안)

### 7.1 주요 컬렉션

#### Users (사용자)
```javascript
{
  _id: ObjectId,
  studentId: String,      // 학번
  password: String,       // 암호화된 비밀번호
  name: String,          // 이름
  department: String,    // 학과
  grade: Number,         // 학년
  email: String,
  createdAt: Date
}
```

#### Courses (강의 정보)
```javascript
{
  _id: ObjectId,
  courseCode: String,    // 과목코드
  courseName: String,    // 과목명
  professor: String,     // 교수명
  credit: Number,        // 학점
  type: String,          // 전공/교양/자유
  department: String,    // 개설학과
  semester: String       // 개설학기
}
```

#### Reviews (강의 리뷰)
```javascript
{
  _id: ObjectId,
  courseId: ObjectId,    // 강의 ID 참조
  userId: ObjectId,      // 작성자 ID 참조
  rating: Number,        // 별점 (1-5)
  difficulty: Number,    // 난이도 (1-5)
  workload: Number,      // 과제량 (1-5)
  tags: [String],        // 태그 배열
  content: String,       // 한줄평
  createdAt: Date
}
```

#### Notices (공지사항)
```javascript
{
  _id: ObjectId,
  title: String,
  content: String,
  category: String,      // 학사/장학/취업/행사
  views: Number,
  createdAt: Date
}
```
