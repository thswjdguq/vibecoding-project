# 마이페이지 및 로고 적용 - 08

## 작업 개요

사용자 프로필 관리를 위한 마이페이지와 비밀번호 변경 페이지를 구현하고, 헤더에 신구대학교 로고 이미지를 적용했습니다. "Simple is Best" 원칙에 따라 깔끔하고 사용하기 쉬운 UI를 구현했습니다.

---

## 주요 변경 사항

### 1. 헤더 로고 적용

#### Before
```tsx
<Link to="/" className="flex items-center gap-3">
  <div className="w-10 h-10 rounded-2xl bg-primary-500 flex items-center justify-center">
    <GraduationCap className="w-6 h-6 text-white" />
  </div>
  <div className="hidden sm:block">
    <div className="text-lg font-bold">신구대학교</div>
    <div className="text-xs">Smart Campus</div>
  </div>
</Link>
```

#### After
```tsx
<Link to="/" className="flex items-center gap-3">
  <img 
    src="/shingu_logo.png" 
    alt="신구대학교 로고" 
    className="h-10"
  />
</Link>
```

**변경 이유:**
- 브랜드 정체성 강화
- UI 간소화
- 모바일/데스크톱 모두 깔끔하게 표시

###2. 라우팅 설정

#### 추가된 라우트
```tsx
<Routes>
  {/* 기존 라우트들 */}
  <Route path="/mypage" element={<MyPage />} />
  <Route path="/mypage/password-change" element={<PasswordChangePage />} />
</Routes>
```

**네비게이션 경로:**
- 헤더 프로필 드롭다운 → 마이페이지
- 마이페이지 → 비밀번호 변경
- 비밀번호 변경 → 마이페이지 (취소)

### 2. PasswordChangePage.tsx

#### 레이아웃 구조
```
     ┌─────────────┐
     │     🔒      │
     │ 비밀번호 변경│
     │ 새로운 비밀번호를│
     │ 설정해주세요    │
     ├─────────────┤
     │ 현재 비밀번호 │
     │ [___________]│
     │             │
     │ 새 비밀번호  │
     │ [___________]│
     │ 💡 8자 이상  │
     │             │
     │ 새 비밀번호 확인│
     │ [___________]│
     │             │
     │[변경하기][취소]│
     └─────────────┘
```

#### 중앙 폼 카드
```tsx
<div className="min-h-[calc(100vh-4rem)] flex items-center justify-center">
  <div className="w-full max-w-2xl bg-white rounded-3xl shadow-sm p-12">
    {/* 헤더 */}
    <div className="text-center mb-8">
      <div className="w-16 h-16 rounded-3xl bg-primary-100">
        <Lock className="w-8 h-8 text-primary-600" />
      </div>
      <h1 className="text-3xl font-bold">비밀번호 변경</h1>
      <p className="text-gray-600">새로운 비밀번호를 설정해주세요</p>
    </div>
    
    {/* 폼 */}
    <form>...</form>
  </div>
</div>
```

**위치:**
- 화면 중앙 (`items-center justify-center`)
- 최대 너비: `max-w-2xl` (672px)
- 패딩: `p-12` (48px)

#### 입력 필드 (3개)
```tsx
{/* 현재 비밀번호 */}
<div>
  <label className="block text-sm font-semibold mb-2">
    현재 비밀번호
  </label>
  <div className="relative">
    <input
      type={showCurrentPassword ? 'text' : 'password'}
      className="w-full px-4 py-4 rounded-2xl border-2 text-lg"
      placeholder="현재 비밀번호를 입력하세요"
    />
    <button type="button" className="absolute right-4 top-1/2">
      {showCurrentPassword ? <EyeOff /> : <Eye />}
    </button>
  </div>
  {errors.currentPassword && <p className="text-error">{errors.currentPassword}</p>}
</div>
```

**필드 특징:**
1. **현재 비밀번호**
   - Show/Hide 토글 (Eye/EyeOff)
   - 검증: 필수 입력

2. **새 비밀번호**
   - Show/Hide 토글
   - 헬퍼 텍스트: "8자 이상, 영문/숫자/특수문자 조합"
   - 검증: 8자 이상, 영문+숫자+특수문자

3. **새 비밀번호 확인**
   - Show/Hide 토글
   - 검증: 새 비밀번호와 일치 여부

#### 비밀번호 검증 로직
```tsx
const validatePassword = (password: string) => {
  if (password.length < 8) return false;
  const hasLetter = /[a-zA-Z]/.test(password);
  const hasNumber = /[0-9]/.test(password);
  const hasSpecial = /[!@#$%^&*(),.?":{}|<>]/.test(password);
  return hasLetter && hasNumber && hasSpecial;
};

const handleSubmit = (e: React.FormEvent) => {
  e.preventDefault();
  const newErrors: {[key: string]: string} = {};

  if (!currentPassword) {
    newErrors.currentPassword = '현재 비밀번호를 입력해주세요';
  }
  
  if (!validatePassword(newPassword)) {
    newErrors.newPassword = '8자 이상, 영문/숫자/특수문자를 포함해야 합니다';
  }
  
  if (newPassword !== confirmPassword) {
    newErrors.confirmPassword = '비밀번호가 일치하지 않습니다';
  }
  
  // ...
};
```

#### 액션 버튼
```tsx
<div className="flex flex-col md:flex-row gap-4">
  {/* 변경하기 - 연두색 강조 */}
  <button
    type="submit"
    className="flex-grow py-4 px-8 bg-primary-500 text-white rounded-2xl hover-lift font-semibold text-lg"
  >
    변경하기
  </button>
  
  {/* 취소 - 텍스트 버튼 */}
  <button
    type="button"
    onClick={() => navigate('/mypage')}
    className="py-4 px-8 text-gray-600 hover:text-gray-900 font-semibold text-lg"
  >
    취소
  </button>
</div>
```

**버튼 스타일:**
- 변경하기: 크고 확실한 버튼 (`bg-primary-500`, `hover-lift`)
- 취소: 작은 텍스트 버튼 (배경 없음)

## 사용자 경험 (UX)

### MyPage
**강점:**
- 한눈에 모든 정보 확인
- Read-only로 실수 방지
- 비밀번호 변경 버튼 명확

**사용 시나리오:**
1. 프로필 드롭다운 클릭
2. "마이페이지" 선택
3. 정보 확인
4. 필요시 비밀번호 변경

### PasswordChangePage
**강점:**
- 집중된 UI (중앙 카드)
- Show/Hide로 입력 확인 편리
- 실시간 검증 피드백
- 명확한 액션 버튼

**사용 시나리오:**
1. 마이페이지에서 "비밀번호 변경" 클릭
2. 현재 비밀번호 입력
3. 새 비밀번호 입력 (헬퍼 텍스트 참고)
4. 확인 입력
5. "변경하기" 클릭
6. 성공 메시지 후 마이페이지로 이동

**오류 처리:**
- 필수 입력 미입력: 빨간 테두리 + 에러 메시지
- 비밀번호 형식 오류: 검증 실패 메시지
- 비밀번호 불일치: 명확한 에러 표시
